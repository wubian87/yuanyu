# 原语言使用指南

> 从零开始，用原语言描述你的软件

---

## 什么是原语言？

原语言是一种**描述语言**，不是编程语言。

你不需要写代码，只需要用结构化的方式描述：
- 你的软件**管理什么信息**
- 用户**能做什么操作**
- 界面**呈现什么样子**
- 有什么**边界和约束**

然后把这个描述交给 AI，它会帮你生成代码。

---

## 快速上手（3 分钟）

### 第 1 步：创建一个 `.yaml` 文件

在任意位置创建一个文件，比如 `我的应用.yaml`

### 第 2 步：写四层描述

```yaml
名称: 待办清单

信息:
  待办:
    内容: string
    完成: boolean
    创建时间: datetime

操作:
  - 添加待办
  - 标记完成
  - 删除待办

呈现:
  首页: 待办列表 + 快速添加框

边界:
  守: [本地存储, 离线可用]
  未: [多人协作, 云同步]
```

### 第 3 步：交给 AI

把这个文件发给任意 AI 编程工具（Cursor、Copilot、Trae 等），说：

> 请根据这个原语言描述，生成一个 [Web/iOS/Android/桌面] 应用

AI 会根据你的描述生成完整代码。

---

## 四层结构详解

### 信息层：是什么

定义你的软件管理什么数据。

```yaml
信息:
  用户:
    名称: string
    头像: image
    积分: number
  
  商品:
    名称: string
    价格: number
    库存: number
```

**常用类型：**

| 类型 | 说明 | 示例 |
|------|------|------|
| `string` | 文本 | 名称、描述 |
| `number` | 数字 | 金额、数量 |
| `boolean` | 是/否 | 是否完成 |
| `datetime` | 时间 | 创建时间 |
| `image` | 图片 | 头像、封面 |
| `[选项1, 选项2]` | 枚举 | 状态、类型 |

### 操作层：做什么

定义用户能执行的动作。

```yaml
操作:
  - 注册
  - 登录
  - 浏览商品
  - 加入购物车
  - 下单
  - 支付
```

### 呈现层：见什么

定义界面长什么样。

```yaml
呈现:
  首页: 推荐商品瀑布流 + 搜索栏
  详情页: 大图 + 价格 + 购买按钮
  购物车: 商品列表 + 总价 + 结算
```

### 边界层：守什么

定义必须做的约束，和暂时不做的功能。

```yaml
边界:
  守:
    - 支付必须验证身份
    - 库存不足不能下单
  未:
    - 国际化多语言
    - 社交分享
```

## 与 AI 协作的技巧

1. **先写描述，再生成** — 不要边写边生成
2. **分模块生成** — 复杂应用拆成多个 `.yaml`
3. **迭代修改** — 改描述，不要直接改代码
4. **指定技术栈** — 告诉 AI 用什么框架
